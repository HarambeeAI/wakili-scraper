  <action>
    Create `worrylesssuperagent/src/hooks/useAgentMarketplace.ts`.

    The hook takes `{ userId: string }` and returns `{ catalog, activeIds, activateAgent, deactivateAgent, isLoading }`.

    On mount, run two parallel Supabase queries:
    ```typescript
    const [{ data: types }, { data: userAgents }] = await Promise.all([
      supabase.from('available_agent_types').select('id, display_name, description, skill_config') as any,
      supabase.from('user_agents').select('agent_type_id').eq('user_id', userId).eq('is_active', true) as any,
    ]);
    setCatalog((types ?? []) as CatalogAgent[]);
    setActiveIds(new Set(((userAgents ?? []) as { agent_type_id: string }[]).map(a => a.agent_type_id)));
    ```

    activateAgent(agentTypeId: string):
    1. Optimistically add to activeIds and set isLoadingId state (disable button during flight)
    2. INSERT into user_agents: `supabase.from('user_agents').insert({ user_id: userId, agent_type_id: agentTypeId })`
    3. On error with code '23505': show toast.error('Agent already in your team') and remove from activeIds
    4. On other error: rollback activeIds, show toast.error('Failed to add agent')
    5. On success: call onAgentChange() via ref (explained below) + show toast.success('Agent added to your team')

    deactivateAgent(agentTypeId: string):
    1. Optimistically remove from activeIds
    2. UPDATE user_agents SET is_active = false where user_id = userId AND agent_type_id = agentTypeId
    3. On error: rollback, show toast.error('Failed to remove agent')
    4. On success: call onAgentChange() + show toast.success('Agent removed from team')

    IMPORTANT: onAgentChange is passed as a prop to the hook. Store it in a ref (useRef) so the callbacks do not need to be in their own dependency arrays (avoids stale closure):
    ```typescript
    const onAgentChangeRef = useRef(onAgentChange);
    useEffect(() => { onAgentChangeRef.current = onAgentChange; }, [onAgentChange]);
    ```

    Use separate isLoadingId state (string | null) to track which agent is currently being activated/deactivated. This disables the button during the async operation to prevent double-clicks.

    Type casts: Add `// as any` with comments where agent_workspaces / available_agent_types are not in generated types.
  </action>
