   - Read `REDIS_URL` from `process.env.REDIS_URL`
   - Throw `Error('REDIS_URL environment variable is required')` if missing
   - Detect TLS: if `url.startsWith('rediss://')`, pass `tls: {}` to IORedis options
   - Always pass `maxRetriesPerRequest: null` (required by BullMQ)
   - Always pass `enableReadyCheck: false` (prevents startup delay)
   - Export a `createRedisConnection()` function (NOT a singleton -- BullMQ requires separate connections for Queue and Worker, per research anti-pattern)

```typescript
import IORedis from 'ioredis';

export function createRedisConnection(): IORedis {
  const url = process.env.REDIS_URL;
  if (!url) throw new Error('REDIS_URL environment variable is required');
  return new IORedis(url, {
    tls: url.startsWith('rediss://') ? {} : undefined,
    maxRetriesPerRequest: null,
    enableReadyCheck: false,
  });
}
```

Note: Use `createRedisConnection()` (factory, creates new instance each call) NOT a singleton. BullMQ requires separate IORedis instances for Queue vs Worker. See research anti-pattern "Do not share a single IORedis instance between Queue and Worker."
