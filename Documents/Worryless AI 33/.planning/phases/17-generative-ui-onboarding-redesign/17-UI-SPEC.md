## Color

All values are CSS custom properties using HSL. Dark mode handled by `.dark` class.

| Role | Light Mode HSL | Dark Mode HSL | Usage |
|------|---------------|---------------|-------|
| Dominant (60%) | 240 4% 95% | 240 5% 10% | Page background, chat scroll area background |
| Secondary (30%) | 0 0% 98% | 240 3% 15% | Agent chat card surface, thread list sidebar panel, onboarding wizard card, HITL card background |
| Accent (10%) | 258 89% 66% | 255 91% 76% | See reserved list below |
| Destructive | 0 72% 50% | 0 84% 60% | Reject button in HITL approval card only |
| Muted | 240 4% 83% | 240 3% 46% | Assistant message bubble background, streaming skeleton, empty state icon opacity |

Accent (`hsl(var(--primary))`) reserved exclusively for:
1. User message bubble background
2. "Send" button fill
3. Active agent name in sidebar (left border + text)
4. Streaming delta cursor dot
5. Business-stage selected option border + check indicator
6. Primary CTA button fill in onboarding ("Continue", "Accept Team")
7. HITL Approve button fill

No other elements may use the accent color.

Source: `worrylesssuperagent/src/index.css` `:root` and `.dark` blocks.

## Screen-by-Screen Interaction Contract

### AgentChatView (GUI-01, GUI-08, GUI-09, GUI-10)

Layout: two-column. Left: `ThreadListSidebar` (240px fixed, collapsible to 0). Right: chat column fills remaining width.

Chat column structure (top to bottom):
1. **Agent header bar** — 56px height, contains agent avatar (32px), agent display name at Heading size, active status dot (green 8px). No other controls.
2. **Message scroll area** — `ScrollArea` from shadcn, `flex-1`, `overflow-y-auto`. Padding: 16px horizontal, 8px vertical between messages.
3. **Streaming tool indicator** — appears above input during SSE stream. `ToolIndicator` pill: `bg-muted`, `rounded-full`, 12px label text, lucide icon matching tool name, `animate-pulse` while active.
4. **Input bar** — 64px min-height, `border-t`, padding 16px. Contains: Paperclip attach button (icon only, 44px), Textarea (flex-1, auto-resize max 5 rows), Send button (primary, 44px square icon-only).

Message bubble contract:
- User: `bg-primary text-primary-foreground`, `rounded-lg rounded-br-none`, max-width 72%, right-aligned, padding 12px 16px.
- Assistant: `bg-muted text-foreground`, `rounded-lg rounded-bl-none`, max-width 80%, left-aligned, padding 12px 16px. Agent name label (12px semibold, muted-foreground) + agent icon above bubble content.
- Bubble spacing: 8px gap between consecutive same-role messages, 16px gap on role change.

Streaming state: assistant bubble appears immediately with `StreamingCursor` (3px dot pulsing at primary color). Text fills in as deltas arrive. Cursor disappears on stream end.

Loading state (pre-stream): single skeleton bubble — `Skeleton` component, height 48px, width 60%, left-aligned, `animate-pulse`.

### HITLApprovalCard (GUI-06)

Structure:
- Card with `border-2 border-primary/30`, `shadow-sm`, `bg-card`, `rounded-lg`, padding 16px.
- Header: warning icon (lucide `AlertTriangle`, amber `text-amber-500`), Heading-size title "Approval Required", Label-size subtitle describing the pending action.
- Body: description of what will happen if approved (body text, 14px).
- Footer: 3 buttons in a row with 8px gap.
  - Approve: `variant="default"` (primary fill), label "Approve", min-width 80px, min-height 44px.
  - Reject: `variant="destructive"`, label "Reject", min-width 80px, min-height 44px.
  - Discuss: `variant="outline"`, label "Discuss", min-width 80px, min-height 44px.

States: pending (all 3 buttons active), approved (card dims to 60% opacity, "Approved" badge replaces buttons, green check icon), rejected (card dims to 60% opacity, "Rejected" badge replaces buttons, red X icon).

### ThreadListSidebar (GUI-10)

Width: 240px. Background: `bg-sidebar` (card-level surface). Border-right: `border-r border-border`.

Header: "Conversations" label (12px semibold uppercase muted), plus `+` icon button (new thread), 8px padding.

Thread item (48px height):
- Agent icon (16px, muted) + thread title truncated at 1 line (14px regular) + relative timestamp (12px, muted-foreground, right-aligned).
- Active thread: `bg-primary/10 text-primary rounded-md`.
- Hover: `bg-muted/60`.
- Empty state: "No past conversations" (12px muted-foreground, centered).

Collapse behavior: on mobile (< 768px), sidebar slides off-screen. Toggle button: `ChevronLeft` / `ChevronRight` lucide icon, 24px, positioned at sidebar edge.

### Onboarding Redesign (ONB-01 through ONB-06)

The existing `ConversationalOnboarding.tsx` is extended with one new step and one modified step. The step sequence becomes:

1. welcome
2. business_name
3. website
4. **business_stage** ← NEW (ONB-01, ONB-02)
5. industry
6. location
7. description
8. meet_team
9. validator_personal_assistant
10. validator_accountant
11. validator_marketer
12. validator_sales
13. agent_team_selector (ONB-03) — already exists, now uses LangGraph CoS graph
14. integration_setup ← NEW (ONB-04)
15. briefing (ONB-05) — now renders real CoS chat message
16. push_opt_in

**BusinessStageSelector** (step: `business_stage`):

Layout: centered card, 100% width max-width 480px. Headline (Display, 28px semibold): "What stage is your business at?" Body (14px regular muted): "We'll build the right AI team and workflow for your stage."

Three option cards (stacked, 8px gap between):
- Starting: "Just getting started — building my first product or service"
- Running: "Operating — have customers, need to manage and grow"
- Scaling: "Growing fast — multiple team members, higher complexity"

Each option card: `border-2 border-border`, `rounded-lg`, padding 16px, label (14px semibold), description (12px regular muted). Selected state: `border-2 border-primary`, primary check icon (16px) top-right corner.

Stage stored to `profiles.business_stage` (TEXT, values: `starting` / `running` / `scaling`).

**integration_setup** step:

Heading: "Connect Your Tools" (Heading 20px semibold). Two optional integration tiles:
- Google Workspace (Gmail + Calendar + Drive): `Button variant="outline"` "Connect Google Account", OAuth redirect.
- Social Browser Session (Marketer): `Button variant="outline"` "Set Up Browser Login", opens browser popup.

Each tile: 80px height, icon (32px) left, title (14px semibold) + subtitle (12px muted), button right-aligned. Skip link below tiles: "I'll do this later" (12px muted-foreground, underline on hover).

**briefing** step (modified):

Replaces static briefing animation. Shows `AgentChatView` restricted to Chief of Staff thread, with the initial briefing message pre-loaded. Progress bar reaches 100% as the SSE stream completes. CTA button "Go to Dashboard" appears after stream ends.

## Accessibility Contract

- All interactive elements have accessible labels. Icon-only buttons have `aria-label`.
- `AgentChatView` scroll area has `role="log" aria-live="polite"` for screen readers.
- HITL approval card has `role="alertdialog" aria-labelledby` pointing to card title.
- Business stage selector uses `role="radiogroup"` with each option as `role="radio"`.
- Keyboard navigation: Tab order in chat: textarea → send → attach. In HITL card: approve → discuss → reject.
- Focus is moved to the message input after each send (`.focus()` on textarea ref).
- Color contrast: all text on muted background meets WCAG AA (4.5:1 minimum).
