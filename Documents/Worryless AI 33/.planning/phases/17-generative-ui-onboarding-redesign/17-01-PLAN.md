    3. Hook implementation details:
       - Uses `useLangGraphFlag()` to determine endpoint
       - `sendMessage`:
         a. Adds user message with `id: crypto.randomUUID()`, `role: "user"`, optimistically
         b. Creates assistant placeholder message with `isStreaming: true`, empty content
         c. Fetch business_stage from Supabase profile BEFORE the POST call:
            ```typescript
            const { data: profile } = await supabase
              .from("profiles")
              .select("business_stage")
              .eq("id", userId)
              .single();
            ```
            Then POST to `${supabaseUrl}/functions/v1/langgraph-proxy/invoke/stream` with body:
            `{ message: text, user_id: userId, thread_id: activeThreadId, agent_type: agentType, business_context: { business_stage: profile?.business_stage ?? null } }`
            and `Authorization: Bearer ${session.access_token}`.
            This ensures ONB-06 requirement is met -- the agent receives business_stage to shape interactions (see RESEARCH.md Pitfall 5).
         d. If `useLangGraph` is false: fallback to legacy `getChatEndpoint(false)` with non-streaming POST (returns JSON, parse response field)
         e. SSE reader pattern (transplanted from ChatInterface.tsx lines 253-303):
            - `response.body.getReader()` + `TextDecoder` + buffer split on `\n`
            - Parse `data: ` lines as JSON
            - `delta` event: append content to streaming assistant message
            - `tool_start` event: `setActiveToolName(data.tool_name)`
            - `ui_components` event: attach `data.components` to current assistant message's `uiComponents` field
            - `pending_approvals` event: `setPendingApprovals(prev => [...prev, ...data.approvals])`
            - `done` event: set `isStreaming=false`, `setActiveToolName(null)`, store `activeThreadId` if null, update message metadata from `data.metadata`
            - `error` event: toast error, set `isStreaming=false`
         f. On fetch/stream error: show destructive toast "Something went wrong. Your message was not sent -- please try again." and remove the optimistic messages

       - `approveHITL`:
         a. POST to `${supabaseUrl}/functions/v1/langgraph-proxy/invoke/resume` with `{ thread_id: threadId, approved, feedback }`
         b. On success: remove the matching PendingApproval, add status message to chat
         c. On failure: toast error

       - Thread loading (useEffect on mount):
         a. GET `${supabaseUrl}/functions/v1/langgraph-proxy/threads/${userId}?agent_type=${agentType}` with auth header
         b. On success: `setThreads(data.threads)`
         c. If threads exist and no activeThreadId: set activeThreadId to most recent thread

       - `startNewThread`: sets `activeThreadId` to null, clears messages

       - When `activeThreadId` changes (and is non-null): load thread state via GET `/langgraph-proxy/threads/${userId}/${activeThreadId}` and populate messages from state.values.messages
