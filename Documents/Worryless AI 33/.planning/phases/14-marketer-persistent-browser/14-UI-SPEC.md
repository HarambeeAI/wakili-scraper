## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline label gaps |
| sm | 8px | Compact element spacing, card internal gaps, icon-to-text gaps |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding, card header padding, dialog content padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, empty state vertical padding |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Touch targets for platform icon buttons (Instagram, X, LinkedIn, TikTok): minimum 44px height — accessibility requirement for mobile
- Browser session status indicator dot: 8px diameter (not on scale, but decorative only)
- Image aspect-ratio cards: `aspect-video` (16:9) for post preview thumbnails, `aspect-square` (1:1) for generated image assets — source: existing MarketerAgent.tsx

5. Active tab indicator in Tabs component

## Interaction States

### Browser Session Status Panel

| State | Visual | Interaction |
|-------|--------|-------------|
| Session valid | Green dot (8px) + "Connected" badge | No action needed |
| Session expired | Red dot (8px) + "Session expired" badge + "Re-login" Button (outline variant) | Button triggers re-login guidance dialog |
| Checking | Skeleton pulse on badge area | Auto-resolves within 15s timeout |
| Never logged in | Gray dot + "Not connected" badge + "Set up {Platform}" Button (primary) | Opens first-run login guidance dialog |

### Publish Post HITL Flow

| Step | Visual |
|------|--------|
| 1. Approval request surface | `PublishApprovalCard` appears inline with post preview, platform badge, estimated publish time |
| 2. User clicks "Approve" | Button enters loading state (Loader2 spinner), card shows "Publishing..." text |
| 3. Success | Card replaced by success toast via Sonner: "Post published to {Platform}" + green icon |
| 4. User clicks "Reject" | AlertDialog: "Cancel publish? This will not delete the scheduled post." Confirm → card dismissed with muted badge "Publish cancelled" |

### Image Generation Loading

| State | Visual |
|-------|--------|
| Idle | Sparkles icon, "Generate" label |
| Generating | Loader2 animate-spin icon, button disabled, placeholder Skeleton in image slot (aspect-square) |
| Complete | Generated image fades in (no animation needed beyond browser default), Sonner toast: "Image generated" |
| Failed | Sonner toast (destructive): "Image generation failed. Try again." |

### Content Calendar Post Card

| State | Visual |
|-------|--------|
| Draft | `bg-muted text-muted-foreground` badge, no schedule line |
| Scheduled | `bg-blue-500/10 text-blue-600` badge, Calendar icon + scheduled datetime |
| Published | `bg-green-500/10 text-green-600` badge, analytics data shown (likes, comments) |
| Hover | `hover:scale-105 hover:-translate-y-1 transition-all duration-200` — matches existing metric card pattern |

## Phase-Specific Interaction Rules

### Platform Tab Pattern

The `BrowserSessionPanel` uses a vertical list (not tabs) for platforms to show all session statuses at a glance. Each row: platform icon (lucide or inline SVG) + platform name + status badge + optional action button. No nested tabs within this panel.

### Image Preview Aspect Ratios

- Post thumbnails in ContentCalendarGrid: `aspect-video` (16:9) — matches existing MarketerAgent.tsx
- Generated images in GeneratedImageGrid: `aspect-square` (1:1) — matches Gemini default output (1:1 for IG square)
- Never stretch or crop images with `object-fill`; always use `object-cover`

### Post Content Truncation

- Post card content preview: `line-clamp-4` (existing pattern from MarketerAgent.tsx)
- HITL approval card preview: `line-clamp-3` to preserve card compactness
- Asset content in assets grid: `line-clamp-3`
- No "Read more" expansion needed in Phase 14 — full content is editable via the Create Post dialog

### Marketer Tools Node — No Frontend for Classification

The `classifyMarketerRequest` regex classification runs server-side only. No frontend UI represents the classification. The user simply types in a chat interface (Phase 17 concern) or uses the Create Post dialog. Phase 14 UI concerns only the static panel views.

### Analytics Bar Chart (PostAnalyticsCard)

Use a simple `Progress` component (shadcn) for relative performance visualization — no Recharts required in Phase 14. Recharts is deferred to Phase 17 (Generative UI). Show: likes count, comments count, reach count, and a "Performance vs. average" bar only.

## Accessibility Requirements

- All platform icon buttons must have `aria-label="{Platform} session status"` — icons alone are not accessible
- Approval card Approve/Reject buttons: minimum 44px height touch target
- Session status dots: supplemented by text badge — color is not the only indicator (WCAG 1.4.1)
- Loading states: `aria-busy="true"` on generating buttons; `aria-label="Generating content"` on Loader2 spinner wrapper
- AlertDialog for destructive actions: focus traps correctly via Radix Dialog primitive (shadcn already handles this)
- Post card image: `alt="Post image for {platform} — {date}"` or `alt="Generated brand image: {title}"`
- Empty state icons: `aria-hidden="true"` — decorative only
