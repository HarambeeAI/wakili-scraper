**1. `readEmails(userId: string, maxResults?: number): Promise<{ emails: EmailMessage[]; count: number }>`** (PA-01)

- Return `{ emails: [...], count: emails.length }` or `{ emails: [], count: 0 }` if empty
- Use the exact code from 15-RESEARCH.md "Gmail Read Emails" section

**2. `triageInbox(userId: string): Promise<{ triaged: TriagedEmail[]; summary: string }>`** (PA-02)

- If empty, return `{ triaged: [], summary: "Your inbox is clear -- no new emails." }`
- Call `callLLMWithStructuredOutput` with schema description: `'{ "emails": [{ "id": "string", "urgency": "urgent|high|normal|low", "topic": "string", "suggestedAction": "respond|delegate|archive|follow_up", "reason": "string" }] }'`
- System prompt: "Categorize each email by urgency and topic. Consider sender importance, subject line, and content snippet."
- Map LLM output to TriagedEmail array, matching email subjects from readEmails result
- Summary string from UI-SPEC: `"{urgentCount} urgent, {highCount} high priority, {normalCount} normal, {lowCount} low priority"`

**3. `draftEmailResponse(userId: string, emailId: string, instructions?: string): Promise<DraftedEmail>`** (PA-03)
- Call `getGoogleClient(userId)`, then fetch full email with `gmail.users.messages.get({ userId: "me", id: emailId, format: "full" })`
- Extract subject, from, body from the email payload
- Call `callLLM` with system prompt: "You are drafting an email response. Match the user's communication style: professional but concise. Draft a complete response to the email below."
- User message: the email content + any user instructions
- Return `{ to: fromAddress, subject: "Re: " + originalSubject, body: llmResult.content, threadId, inReplyTo: emailId }`

**4. `sendEmail(userId: string, agentType: string, email: DraftedEmail): Promise<{ sent: boolean; messageId?: string; message: string }>`** (PA-04)
- Call `interruptForApproval({ action: "send_email", agentType: agentType as any, description: \`Send email to ${email.to}: "${email.subject}"\`, payload: email })`
- If not approved, return `{ sent: false, message: "Email send cancelled by user." }`
- If approved, call `getGoogleClient(userId)`, construct MIME message (To, Subject, Content-Type headers + body), base64url encode it
- `gmail.users.messages.send({ userId: "me", requestBody: { raw: base64Message, threadId: email.threadId } })`
- Return `{ sent: true, messageId: result.data.id, message: \`Email sent to ${email.to}\` }`

**1. Create `src/tools/pa/calendar-tools.ts`** with 4 exported functions:

**`listCalendarEvents(userId: string, timeMin?: string, timeMax?: string): Promise<{ events: CalendarEvent[]; count: number }>`** (PA-05)
- Call `getGoogleClient(userId)`, then `google.calendar({ version: "v3", auth })`
- Default timeMin to start of today (ISO), timeMax to end of today + 7 days
- `calendar.events.list({ calendarId: "primary", timeMin, timeMax, singleEvents: true, orderBy: "startTime", maxResults: 50 })`
- Map to CalendarEvent type. If empty, message: `"No events found for {dateRange}."`

**`createCalendarEvent(userId: string, agentType: string, eventDetails: CreateEventInput): Promise<{ created: boolean; conflict?: boolean; busySlots?: BusySlot[]; eventId?: string; htmlLink?: string; message: string }>`** (PA-06)
- Use the exact code from 15-RESEARCH.md "Calendar FreeBusy Check + Event Creation"
- Check `calendar.freebusy.query()` first. If busy, return conflict info with message from UI-SPEC: `"Conflict detected: you have {count} event(s) during this time."`
- Call `interruptForApproval({ action: "create_calendar_event", agentType, description: \`Create calendar event: "${eventDetails.summary}" from ${eventDetails.startTime} to ${eventDetails.endTime}\`, payload: eventDetails })`
- If not approved, return `{ created: false, message: "Event creation cancelled by user." }`
- `calendar.events.insert({ calendarId: "primary", sendUpdates: "all", requestBody: { summary, description, location, start: { dateTime }, end: { dateTime }, attendees } })`
- Return `{ created: true, eventId, htmlLink, message: "Calendar event created." }`

**`detectCalendarConflicts(userId: string, date?: string): Promise<ConflictResult>`** (PA-09)
- Call `listCalendarEvents(userId, startOfDay, endOfDay)` for given date (default today)
- Sort events by start time, iterate checking for overlaps: if event[i].end > event[i+1].start, mark as conflict
- Call `callLLM` with conflict pairs asking for resolution suggestions
- Return `{ hasConflict, busySlots: conflictingPairs, suggestions }`

**`analyzeTimeAllocation(userId: string, days?: number): Promise<TimeAllocation>`** (PA-10)
- Call `listCalendarEvents(userId, startDate, endDate)` for the past N days (default 7)
- Calculate total hours in range, meeting hours (sum of event durations), focus hours = total business hours - meeting hours
- Calculate percentages, find busiest day by meeting count
- Return TimeAllocation with message from UI-SPEC: `"{meetingHours}h in meetings ({meetingPercent}%), {focusHours}h focus time ({focusPercent}%)"`

**2. Create `src/tools/pa/meeting-tools.ts`** with 1 exported function:

**`prepareMeetingBrief(userId: string, eventId: string): Promise<MeetingBrief>`** (PA-07)
