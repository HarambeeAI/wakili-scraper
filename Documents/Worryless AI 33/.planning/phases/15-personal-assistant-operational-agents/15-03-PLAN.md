**Create `src/tools/customer-support/ticket-tools.ts`** with 4 functions:

1. **`createTicket(input: CreateTicketInput): Promise<{ ticketId: string; message: string }>`**
   - INSERT into public.support_tickets (user_id, customer_name, customer_email, subject, description, priority, category) VALUES ($1-$7) RETURNING id
   - Message from UI-SPEC: `"Support ticket created: #{ticketId} -- \"{subject}\""`

2. **`listTickets(userId: string, status?: string): Promise<{ tickets: SupportTicketRow[]; count: number; message: string }>`**
   - SELECT * FROM public.support_tickets WHERE user_id = $1 [AND status = $2] ORDER BY created_at DESC LIMIT 50
   - Empty message from UI-SPEC: `"No support tickets found."`

3. **`updateTicket(userId: string, ticketId: string, updates: { status?: string; resolution?: string; priority?: string }): Promise<{ updated: boolean; message: string }>`**
   - UPDATE public.support_tickets SET status=$3, resolution=$4, priority=$5, updated_at=now() [, resolved_at=now() if status='resolved'] WHERE id=$2 AND user_id=$1
   - If status is 'resolved', set resolved_at. Message: `"Ticket #{ticketId} marked as resolved."`

4. **`searchKBAndDraftResponse(userId: string, query: string): Promise<{ kbResults: Array<{source, content, similarity}>; draftResponse: string | null; message: string }>`**
   - Use the exact code from 15-RESEARCH.md "Customer Support KB RAG Search" section
   - Call `ragRetrieveByText(userId, query, 5, "customer_support")`
   - If empty, return with message from UI-SPEC: `"No knowledge base articles found. Add business documentation to enable grounded responses."`
   - Call `callLLM` to draft response grounded in KB results
   - Message: `"Draft response based on {resultCount} knowledge base article(s)."`

**Create `src/tools/customer-support/health-tools.ts`** with 2 functions:

5. **`scoreCustomerHealth(userId: string, customerName: string): Promise<HealthScore>`**
   - SELECT COUNT(*), MAX(created_at), AVG(CASE WHEN resolved_at IS NOT NULL THEN EXTRACT(EPOCH FROM (resolved_at - created_at)) END) FROM support_tickets WHERE user_id=$1 AND customer_name=$2
   - Calculate score 0-100: fewer tickets + faster resolution = higher score
   - Message from UI-SPEC: `"Customer health score: {score}/100"`

6. **`detectChurnRisk(userId: string): Promise<{ risks: ChurnRisk[]; message: string }>`**
   - SELECT customer_name, COUNT(*) as ticket_count, MAX(created_at) as last_ticket FROM support_tickets WHERE user_id=$1 AND created_at > NOW() - INTERVAL '30 days' GROUP BY customer_name HAVING COUNT(*) >= 3
   - Classify risk: >=5 tickets = high, >=3 = medium
   - If no risks, message from UI-SPEC: `"No churn risk indicators found across {customerCount} customers."`
   - If risks, message: `"Churn risk detected for {customerName}: {riskLevel} ({reason})."`

**Create `src/tools/legal/contract-tools.ts`** with 4 functions:

1. **`createContract(userId: string, title: string, counterparty: string, contractType?: string, value?: number, startDate?: string, endDate?: string, renewalDate?: string): Promise<{ contractId: string; message: string }>`**
   - INSERT into public.contracts RETURNING id
   - Message from UI-SPEC: `"Contract created: \"{title}\" with {counterparty}"`

2. **`listContracts(userId: string, status?: string): Promise<{ contracts: ContractRow[]; count: number; message: string }>`**
   - SELECT * FROM public.contracts WHERE user_id=$1 [AND status=$2] ORDER BY created_at DESC
   - Empty message from UI-SPEC: `"No contracts on file."`

3. **`reviewContract(userId: string, contractId: string, contractText: string): Promise<ContractReview>`**
   - Call `callLLMWithStructuredOutput` with schema: `'{ "riskFlags": [{ "severity": "high|medium|low", "description": "string", "clause": "string" }], "keyTerms": { "parties": "string", "payment": "string", "termination": "string", "ip": "string", "liability": "string" }, "recommendation": "string" }'`
   - System prompt: "Review this contract for legal risks. Identify risk flags by severity, extract key terms, and provide a recommendation. Caveat: This is not legal advice."
   - UPDATE contracts SET risk_flags = $riskFlags, key_terms = $keyTerms WHERE id=$contractId AND user_id=$userId
   - Message: `"Contract review complete. {riskCount} risk flag(s) identified."` or `"No risk flags identified."`

4. **`draftTemplate(userId: string, templateType: string, parties?: string): Promise<{ template: string; message: string }>`**
   - Call `callLLM` with prompt specifying template type (NDA, MSA, SOW, etc.) and parties
   - Message from UI-SPEC: `"Contract template drafted: \"{title}\""`

**Create `src/tools/legal/compliance-tools.ts`** with 2 functions:

5. **`contractCalendar(userId: string, daysAhead?: number): Promise<{ renewals: RenewalAlert[]; message: string }>`**
   - SELECT id, title, counterparty, renewal_date, value FROM contracts WHERE user_id=$1 AND renewal_date IS NOT NULL AND renewal_date <= NOW() + INTERVAL '$daysAhead days' ORDER BY renewal_date
   - Default daysAhead = 60
   - Message: `"{count} contract(s) due for renewal within {days} days."` or `"No contract renewals due in the next {days} days."`

6. **`monitorRegulatory(userId: string, industry?: string): Promise<{ updates: Array<{ topic: string; summary: string; relevance: string }>; message: string }>`**
   - Call `callLLM` to identify relevant regulatory topics for the user's industry
   - Return structured updates. This is an LLM-only tool (no external API).
