    The component accepts:
    ```typescript
    interface PushOptInBannerProps {
      userId: string;
      onDismiss: () => void;   // Called on both accept and skip
    }
    ```

    Implementation rules:
    1. Import `usePushSubscription` from `@/hooks/usePushSubscription`
    2. Import `Button` from `@/components/ui/button`
    3. At the top of the component body, return null (render nothing) if:
       - `!('PushManager' in window)` — browser does not support push
       - `Notification.permission === 'denied'` — user already blocked; do not show a useless UI
    4. If `isSubscribed` is true, call `onDismiss()` inside a `useEffect([isSubscribed])` and return null — auto-dismiss if already subscribed
    5. Render a card-style UI with:
       - Icon: Bell or BellRing from lucide-react
       - Heading: "Stay informed on urgent findings"
       - Body: "Get instant push alerts when your AI agents flag something that needs your attention."
       - Two buttons side-by-side:
         - Primary: "Enable notifications" — `onClick={async () => { await subscribe(); onDismiss(); }}` disabled while `isLoading`
         - Ghost: "Skip for now" — `onClick={onDismiss}`
    6. Use Tailwind classes consistent with the dashboard card style: `rounded-xl border bg-card p-6 shadow-sm` for the container, `flex flex-col gap-4` for layout, `flex gap-3 justify-end` for button row.

    Example structure:
    ```tsx
    export function PushOptInBanner({ userId, onDismiss }: PushOptInBannerProps) {
      const { isSubscribed, isLoading, subscribe } = usePushSubscription(userId);

      useEffect(() => {
        if (isSubscribed) onDismiss();
      }, [isSubscribed, onDismiss]);

      if (!('PushManager' in window) || Notification.permission === 'denied') {
        return null;
      }

      return (
        <div className="rounded-xl border bg-card p-6 shadow-sm">
          ...
        </div>
      );
    }
    ```
