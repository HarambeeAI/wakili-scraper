**1. Create `src/tools/accountant/parse-receipt.ts` (ACCT-04):**

CRITICAL: Do NOT use callLLM — it stringifies content arrays into JSON strings, breaking multimodal. Use direct fetch to Lovable AI Gateway.

```typescript
import type { ParsedReceipt } from "./types.js";

const LOVABLE_AI_GATEWAY = "https://ai.gateway.lovable.dev/v1/chat/completions";

// ACCT-04: Parse receipt photo using Gemini multimodal vision.
// Bypasses callLLM because callLLM's messagesToOpenAI() stringifies content arrays,
// which breaks the image_url content format required by the vision model.
export async function parseReceipt(
  base64Image: string,
  mimeType: string,
  userId: string,
): Promise<ParsedReceipt> {
  const apiKey = process.env.LOVABLE_API_KEY;
  if (!apiKey) throw new Error("LOVABLE_API_KEY environment variable is required");

  const response = await fetch(LOVABLE_AI_GATEWAY, {
    method: "POST",
    headers: {
      Authorization: `Bearer ${apiKey}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      model: "google/gemini-3-flash-preview",
      messages: [
        {
          role: "user",
          content: [
            {
              type: "image_url",
              image_url: { url: `data:${mimeType};base64,${base64Image}` },
            },
            {
              type: "text",
              text: "Extract all transaction details from this receipt. Return ONLY valid JSON with: {\"vendor\": \"string\", \"amount\": number, \"currency\": \"string (3-letter code)\", \"date\": \"YYYY-MM-DD\", \"category\": \"food|transport|office|utilities|other\", \"line_items\": [{\"description\": \"string\", \"amount\": number}]}",
            },
          ],
        },
      ],
      temperature: 0.1,
      max_tokens: 1024,
    }),
  });

  if (!response.ok) {
    const errorText = await response.text();
    throw new Error(`Receipt parsing failed (${response.status}): ${errorText}`);
  }

  const data = (await response.json()) as {
    choices?: Array<{ message?: { content?: string } }>;
  };
  let content = data.choices?.[0]?.message?.content ?? "{}";

  // Strip markdown fences if present
  if (content.startsWith("```")) {
    content = content.replace(/^```(?:json)?\n?/, "").replace(/\n?```$/, "");
  }

  return JSON.parse(content) as ParsedReceipt;
}
```

**2. Create `src/tools/accountant/cashflow-tools.ts` (ACCT-05):**

Import `getPool` from `../shared/db.js`, types from `./types.js`.

`calculateCashflowProjection(userId: string): Promise<CashflowProjection[]>`:
- Query 1: Get recent 90-day transaction summary grouped by month:
  ```sql
  SELECT type, SUM(amount) as total
  FROM public.transactions
  WHERE user_id = $1 AND date >= NOW() - INTERVAL '90 days'
  GROUP BY type
  ```
- Query 2: Get pending invoices:
  ```sql
  SELECT SUM(amount) as total
  FROM public.invoices
  WHERE user_id = $1 AND status = 'pending'
  ```
- Calculate monthly averages from 90-day data: avgIncome = total income / 3, avgExpenses = total expenses / 3
- Calculate current cash position: total income - total expenses (from all transactions)
  ```sql
  SELECT COALESCE(SUM(CASE WHEN type='income' THEN amount ELSE -amount END), 0) as cash_balance
  FROM public.transactions
  WHERE user_id = $1
  ```
- Build 3 projections (30d, 60d, 90d):
  - period "30d": startingCash = cash_balance, projectedIncome = avgIncome * 1, projectedExpenses = avgExpenses * 1, projectedBalance = startingCash + projectedIncome - projectedExpenses, pendingInvoices = pending total
  - "60d": multiply by 2
  - "90d": multiply by 3
- Return array of 3 CashflowProjection objects

**3. Create `src/tools/accountant/report-tools.ts` (ACCT-06):**

`generatePLReport(userId: string): Promise<PLReport>`:
- Query 1: Group transactions by month, type, and category for last 3 months:
  ```sql
  SELECT
    to_char(date_trunc('month', date), 'YYYY-MM') AS month,
    type,
    COALESCE(category, 'uncategorized') as category,
    SUM(amount) AS total
  FROM public.transactions
  WHERE user_id = $1
    AND date >= NOW() - INTERVAL '3 months'
  GROUP BY 1, 2, 3
  ORDER BY 1 DESC
  ```
- Query 2: Get invoice receivables summary:
  ```sql
  SELECT status, COUNT(*)::int as count, COALESCE(SUM(amount), 0) AS total
  FROM public.invoices
  WHERE user_id = $1
  GROUP BY status
  ```
- Build PLReport:
  - Group transaction rows by month
  - For each month: sum income rows, sum expense rows, calculate netProfit = income - expenses, build byCategory map
  - Build receivables from invoice summary: { pending: sum where status='pending', overdue: sum where status='overdue', paid: sum where status='paid' }
- Return PLReport
