# Phase 13 — UI Design Contract

> Visual and interaction contract for the Accountant + Sales Rep Agent Tools phase.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Scope Note

Phase 13 is a **backend-first phase**. All new code lives in the LangGraph server
(`worrylesssuperagent/langgraph-server/src/tools/`). There are no new React
components. The UI contract covers:

1. HITL approval cards surfaced by `interruptForApproval` for `chase_overdue_invoice`
   (ACCT-10) and `send_outreach` (SALES-05)
2. Tool-in-progress loading states in the existing `AccountantAgent.tsx` and
   `SalesRepAgent.tsx` panels
3. Toast copy for tool success, tool error, and HITL outcomes
4. Status badge colours for two new pipeline stages (`proposal`, `closed_won`,
   `closed_lost`) added to `SalesRepAgent.tsx`

Everything else (spacing, typography, colour palette, component library) is fully
inherited from the existing design system and must not be overridden.

## Spacing Scale

Inherited from existing system. No new tokens required for Phase 13.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact row padding, icon-to-label gap |
| md | 16px | Card body padding, dialog section spacing |
| lg | 24px | Card header padding, dialog vertical rhythm |
| xl | 32px | Section breaks within agent panels |
| 2xl | 48px | Empty state icon-to-text gap |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- HITL approval card action button row: use `gap-4` (16px) between Approve and Reject
  buttons — standard spacing scale value md.
- Touch target minimum for Approve/Reject buttons: 48px height — use `h-12` (48px),
  which is in the standard set and exceeds the WCAG 44px minimum.

## Typography

Inherited from existing system. No new type roles needed.

| Role | Size | Weight | Line Height | Usage in Phase 13 |
|------|------|--------|-------------|-------------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 | Toast descriptions, HITL card body copy, pipeline row labels |
| Label | 12px (text-xs) | 600 (semibold) | 1.4 | Badge text, HITL action labels, status chip text |
| Heading | 16px (text-base) | 600 (semibold) | 1.2 | HITL card title, dialog title, section headings |
| Display | 24px (text-2xl) | 600 (semibold) | 1.1 | Financial metric values (income, expenses, pending total) |

Active weights: **400 (normal)** and **600 (semibold)** only. The 24px size alone
provides sufficient display-level visual weight without requiring a third weight.

Monospace variant: IBM Plex Mono at 14px weight 400 — used for raw currency amounts
in the HITL invoice chase card (invoice ID, exact amount). Apply `font-mono` class.

## Interaction States

### HITL Approval Card — `chase_overdue_invoice` (ACCT-10)

Rendered inside the existing agent chat or notification panel when `interruptForApproval`
fires. Uses shadcn `Card` + `CardHeader` + `CardContent`.

**Structure:**
```
Card (border, shadow-sm, rounded-lg)
  CardHeader
    Badge ("Action Required" — accent/primary colour, outline variant)
    CardTitle  "Send Payment Reminder"           [Heading: 16px semibold]
    CardDescription  "Invoice #[ID] — [Vendor]" [Body: 14px normal muted]
  CardContent
    Two-column grid (left: invoice details, right: email draft preview)
    Invoice details:
      "Vendor"    [Label]    [Value — font-mono]
      "Amount"    [Label]    $[amount] [Value — font-mono, text-destructive if overdue]
      "Days Overdue"  [Label]  [N] days
    Email draft (read-only Textarea, 6 rows, bg-muted/40)
    Separator (8px vertical)
    Button row (gap-4, justify-end)
      Button variant="outline" size="h-12"   "Reject"    [text-destructive]
      Button variant="default" size="h-12"   "Approve & Send"
```

**States:**
- Default: both buttons enabled, Approve uses `--primary` fill
- Submitting: Approve button shows `<Loader2 className="h-4 w-4 animate-spin mr-2" />` + "Sending..." text; Reject disabled
- Approved: Card replaced by success toast (see Copywriting)
- Rejected: Card dismissed, rejection toast shown

### HITL Approval Card — `send_outreach` (SALES-05)

Same Card structure as above with these differences:

**Structure differences:**
```
CardTitle  "Send Outreach Email"
CardDescription  "To: [Contact Name] at [Company]"
Email draft: Subject line (Input, read-only) + Body (Textarea, 8 rows, read-only)
Button row:
  Button variant="outline" size="h-12"  "Reject"         [text-destructive]
  Button variant="outline" size="h-12"  "Edit & Approve" [text-primary, border-primary]
  Button variant="default" size="h-12"  "Approve & Send"
```

Edit & Approve: makes Subject Input and Body Textarea editable (removes `readOnly`);
relabels button to "Save & Send".

### Tool-In-Progress Loading State

When an agent tool node is executing (graph is in the `accountantTools` or
`salesTools` node), the existing panel shows:

```
<div className="flex items-center gap-2 py-4 text-sm text-muted-foreground">
  <Loader2 className="h-4 w-4 animate-spin text-primary" />
  <span>{toolProgressLabel}</span>
</div>
```

Tool progress labels (exact strings):

| Tool | Label |
|------|-------|
| `list_invoices`, `generate_pl_report` | "Pulling your financials..." |
| `calculate_cashflow_projection`, `forecast_runway` | "Calculating cashflow..." |
| `parse_bank_statement` | "Parsing bank statement..." |
| `parse_receipt` | "Reading receipt..." |
| `generate_leads` | "Finding leads via Apify..." |
| `research_prospect` | "Researching prospect..." |
| `compose_outreach` | "Drafting outreach email..." |
| `analyze_pipeline`, `detect_stale_deals` | "Analysing pipeline..." |
| Any other tool | "Working on it..." |

The progress label is injected into `responseMetadata.toolInProgress` in AgentState
by the tool-execution node and read by the existing panel render path.

## Accessibility Contract

- HITL Approve button: `aria-label="Approve and send [action description]"`
- HITL Reject button: `aria-label="Reject [action description]"`
- HITL card: `role="alertdialog"` when surfaced as a blocking modal; `aria-live="assertive"`
- Loading spinner: `aria-label="Loading"` on the `<Loader2>` icon wrapper
- Status badges: `aria-label="Status: [status value]"` on each `<Badge>`
- Financial figures: wrap in `<span aria-label="[amount] dollars">` for screen readers

Focus order on HITL cards: Badge → Title → Description → Draft preview → Reject → (Edit) → Approve.
