Port the Apify integration from v1 Edge Function `supabase/functions/generate-leads/index.ts` to Node.js.

Import `getPool` from `../shared/db.js`, types from `./types.js`.

`generateLeads(input: GenerateLeadsInput): Promise<LeadRow[]>`:
- Check env: `const APIFY_API_TOKEN = process.env.APIFY_API_TOKEN; if (!APIFY_API_TOKEN) throw new Error("APIFY_API_TOKEN not configured — add your Apify API key in Settings.");`
- Build Apify input payload:
  ```typescript
  const apifyInput: Record<string, unknown> = {
    fetch_count: Math.min(input.fetchCount ?? 20, 20),  // Cap at 20 for LangGraph (shorter timeout than Edge Function)
  };
  if (input.query) apifyInput.company_keywords = [input.query];
  if (input.jobTitle) apifyInput.contact_job_title = [input.jobTitle];
  if (input.location) apifyInput.contact_location = [input.location.toLowerCase()];
  if (input.industry) apifyInput.company_industry = [input.industry.toLowerCase()];
  ```
- Fetch from Apify sync endpoint:
  ```typescript
  const apifyUrl = `https://api.apify.com/v2/acts/code_crafter~leads-finder/run-sync-get-dataset-items?token=${APIFY_API_TOKEN}`;
  const response = await fetch(apifyUrl, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(apifyInput),
  });
  if (!response.ok) throw new Error(`Apify API error: ${response.status}`);
  const data: ApifyLead[] = await response.json();
  ```
- **CRITICAL (dedup):** Upsert to leads table using `ON CONFLICT` on the partial unique index `idx_leads_user_email_unique (user_id, email) WHERE email IS NOT NULL`. For each Apify lead, use:
  ```sql
  INSERT INTO public.leads (user_id, company_name, contact_name, email, phone, website, industry, company_size, location, status, score, notes, source)
  VALUES ($1, $2, $3, $4, $5, $6, $7, $8, $9, 'prospecting', 0, $10, 'apify_leads_finder')
  ON CONFLICT (user_id, email) WHERE email IS NOT NULL
  DO UPDATE SET
    updated_at = NOW(),
    notes = EXCLUDED.notes,
    phone = COALESCE(EXCLUDED.phone, public.leads.phone),
    website = COALESCE(EXCLUDED.website, public.leads.website),
    industry = COALESCE(EXCLUDED.industry, public.leads.industry),
    company_size = COALESCE(EXCLUDED.company_size, public.leads.company_size),
    location = COALESCE(EXCLUDED.location, public.leads.location)
  RETURNING *
  ```
  This ensures repeated calls with the same query do not create duplicate lead rows. Existing leads are updated with any new data from Apify (COALESCE preserves existing non-null values).
  For leads without an email (contact_email is null/empty), use a plain INSERT without ON CONFLICT since the unique index only covers rows WHERE email IS NOT NULL.
- Return the inserted/updated LeadRow[]

`enrichLeadData(userId: string, leadId: string): Promise<LeadRow>`:
- Fetch the lead:
  ```sql
  SELECT * FROM public.leads WHERE id = $1 AND user_id = $2
  ```
- If lead has a website, scrape it via Firecrawl:
  ```typescript
  const FIRECRAWL_API_KEY = process.env.FIRECRAWL_API_KEY;
  if (FIRECRAWL_API_KEY && lead.website) {
    const scrapeResponse = await fetch("https://api.firecrawl.dev/v1/scrape", {
      method: "POST",
      headers: { Authorization: `Bearer ${FIRECRAWL_API_KEY}`, "Content-Type": "application/json" },
      body: JSON.stringify({ url: lead.website, formats: ["markdown"], onlyMainContent: true }),
    });
    if (scrapeResponse.ok) {
      const scrapeData = await scrapeResponse.json();
      pageContent = scrapeData.data?.markdown ?? "";
    }
  }
  ```
- Call `callLLMWithStructuredOutput` to synthesize enrichment data from website content + existing lead data, schema: `'{"description": "string", "products": ["string"], "teamSize": "string", "recentActivity": "string", "enrichedNotes": "string"}'`, temperature: 0.2
- Update lead notes:
  ```sql
  UPDATE public.leads SET notes = $1, updated_at = NOW() WHERE id = $2 AND user_id = $3 RETURNING *
  ```
  Where notes = JSON.stringify(enrichment data) appended to existing notes
- Return updated lead

`sendOutreach(userId: string, outreach: OutreachEmail, recipientEmail: string): Promise<string>`:

- Send via Resend:
  ```typescript
  const response = await fetch("https://api.resend.com/emails", {
    method: "POST",
    headers: { Authorization: `Bearer ${RESEND_API_KEY}`, "Content-Type": "application/json" },
    body: JSON.stringify({
      from: "Worryless AI Team <myteam@worryless.ai>",
      to: [recipientEmail],
      subject: outreach.subject,
      html: outreach.body,
    }),
  });
  ```

**3. Create `src/tools/sales/email-engagement.ts` (SALES-06):**

Import `getPool` from `../shared/db.js`, types from `./types.js`.

`trackEmailEngagement(userId: string, leadId?: string): Promise<EmailEngagement[]>`:
- Query outreach_emails joined with leads:
  ```sql
  SELECT
    oe.id as email_id,
    oe.subject,
    oe.sent_at,
    oe.open_count as opens,
    oe.click_count as clicks,
    oe.replied_at IS NOT NULL as replied,
    l.contact_name as lead_name,
    l.company_name
  FROM public.outreach_emails oe
  JOIN public.leads l ON l.id = oe.lead_id
  WHERE oe.user_id = $1
  ${leadId ? "AND oe.lead_id = $2" : ""}
  AND oe.sent_at IS NOT NULL
  ORDER BY oe.sent_at DESC
  LIMIT 50
  ```
- Map to EmailEngagement[]
- Return results

Note: The open_count and click_count columns are updated by a Resend webhook handler (separate Edge Function, not part of Phase 13 tools). The tool reads whatever data is available. If no webhook is configured yet, all counts will be 0.
