```typescript
// Step 4b: Write morning briefing to CoS LangGraph thread (for use_langgraph users)
const LANGGRAPH_SERVER_URL = Deno.env.get("LANGGRAPH_SERVER_URL");
if (LANGGRAPH_SERVER_URL && profile?.use_langgraph) {
  try {
    // Construct a CoS morning briefing prompt with the digest content
    const briefingPrompt = `Compile the morning briefing from these overnight agent findings:\n\n${digestText}\n\nOrganize by urgency, provide action recommendations, and correlate any cross-agent findings.`;

    const cosThreadId = `proactive:chief_of_staff:${userId}`;

    await fetch(`${LANGGRAPH_SERVER_URL}/invoke`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        message: briefingPrompt,
        user_id: userId,
        thread_id: cosThreadId,
        agent_type: "chief_of_staff",
      }),
    });
    // Fire-and-forget — morning digest notification is the critical path, not the CoS thread
  } catch (cosErr) {
    console.error(`[send-morning-digest] CoS thread write failed for user ${userId} (non-fatal):`, cosErr);
  }
}
```

Key details:
- Only calls LangGraph if `LANGGRAPH_SERVER_URL` env var is set AND `use_langgraph = true` for the user
- Uses deterministic thread ID `proactive:chief_of_staff:${userId}` (same format as proactive-runner)
- Passes the digest text as context in the briefing prompt so CoS can organize and correlate
- Fire-and-forget — failure does NOT block the notification/email path
- The `use_langgraph` flag must be fetched. Add it to the profiles query: change `select("email, timezone, ...")` to also include `use_langgraph`

**proactive-runner update:**

The proactive-runner from Plan 02 handles daily/weekly/monthly/quarterly tiers. Add event-type handling:

When `cadence_tier === 'event'`, the pgmq message also contains an `event_type` field (e.g., 'overdue_invoice', 'stale_deal', 'expiring_contract'). The proactive-runner must:

1. Check for `event_type` in the message payload
2. Look up the event-specific prompt from an inline `EVENT_PROMPTS` map (duplicated from event-detector.ts since Deno cannot import from langgraph-server Node modules)
3. Use the event prompt instead of the cadence tier prompt
4. If event_type is unknown, use a generic fallback: "An event was triggered for your attention. Review your recent activity and surface any actionable findings."

Add to the proactive-runner, alongside the existing HEARTBEAT_PROMPTS:

