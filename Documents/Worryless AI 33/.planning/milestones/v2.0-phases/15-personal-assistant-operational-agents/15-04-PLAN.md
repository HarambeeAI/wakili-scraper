**Create `src/tools/pr/media-tools.ts`** with 3 functions:

1. **`draftPressRelease(userId: string, topic: string, keyPoints: string[]): Promise<{ pressRelease: string; message: string }>`**
   - Call `callLLM` with system prompt: "Draft a professional press release. Include headline, dateline, body (inverted pyramid), quotes placeholder, and boilerplate."
   - Message from UI-SPEC: `"Press release drafted: \"{title}\""`

2. **`monitorMedia(userId: string, businessName: string): Promise<{ mentions: MediaMention[]; message: string }>`**
   - Use Firecrawl search (same pattern as marketer's `monitorBrandMentions`): extract `firecrawlSearch` helper or reimplement the fetch to `https://api.firecrawl.dev/v1/search` with `FIRECRAWL_API_KEY` from env
   - Call `callLLMWithStructuredOutput` to classify sentiment of each result
   - Empty message from UI-SPEC: `"No media mentions found for \"{query}\"."`
   - Result message: `"Found {mentionCount} mention(s) across {sourceCount} source(s). Sentiment: {positiveCount} positive, {neutralCount} neutral, {negativeCount} negative."`

3. **`trackCoverage(userId: string, publication: string, title: string, url?: string, journalist?: string, reach?: number, sentiment?: string): Promise<{ coverageId: string; message: string }>`**
   - INSERT into public.press_coverage RETURNING id
   - Message from UI-SPEC: `"Coverage tracked: \"{title}\" in {publication}"`

**Create `src/tools/pr/sentiment-tools.ts`** with 1 function:

4. **`analyzeSentiment(userId: string): Promise<SentimentAnalysis>`**

   - Calculate overall sentiment and score (-1 to 1 range)
   - Message from UI-SPEC: `"Overall sentiment: {overallSentiment} ({sentimentScore})"`

**Create `src/tools/procurement/supplier-tools.ts`** with 3 functions:

1. **`searchSuppliers(userId: string, query: string): Promise<{ suppliers: SupplierResult[]; message: string }>`**
   - Use Firecrawl search for supplier discovery (same pattern as PR media monitoring)
   - Call `callLLMWithStructuredOutput` to extract structured supplier info from search results
   - Empty message from UI-SPEC: `"No suppliers found for \"{query}\"."`
   - Result message: `"Found {count} potential supplier(s) for \"{query}\"."`

2. **`compareQuotes(userId: string, suppliers: Array<{ name: string; price: number; terms: string; quality: number; leadTime: string }>): Promise<QuoteComparison>`**
   - Score each supplier: normalize price (lower=better), quality (higher=better), parse leadTime
   - Total score = weighted average (price 40%, quality 30%, leadTime 30%)
   - Message from UI-SPEC: `"Quote comparison: {supplierCount} supplier(s) compared. Best value: {bestSupplier} at {bestPrice}."`

3. **`scoreVendor(userId: string, supplierName: string): Promise<VendorScore>`**
   - Query agent_audit_log or procurement history for past interactions (if available)
   - Calculate reliability, price, quality scores 0-100
   - Message from UI-SPEC: `"Vendor score for {supplierName}: {score}/100 (Reliability: {reliability}, Price: {price}, Quality: {quality})"`

**Create `src/tools/procurement/po-tools.ts`** with 1 function:

4. **`createPurchaseOrder(userId: string, agentType: string, input: PurchaseOrderInput): Promise<{ created: boolean; poId?: string; message: string }>`**
   - Call `interruptForApproval({ action: "create_purchase_order", agentType, description: \`Create purchase order: ${input.quantity}x "${input.itemDescription}" from ${input.supplier} for ${input.totalAmount}\`, payload: input })`
   - If not approved, return `{ created: false, message: "Purchase order creation cancelled by user." }`
   - If approved, INSERT into agent_tasks or a dedicated tracking structure. Since no purchase_orders table exists, store as an agent_asset with asset_type='purchase_order'.
   - Message from UI-SPEC: `"Purchase order #{poId} created: {totalAmount} to {supplier}"`
