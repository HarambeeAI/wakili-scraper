# Phase 15 — UI Design Contract

> Visual and interaction contract for Phase 15: Personal Assistant + Operational Agents.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Phase Classification: Backend-Only

Phase 15 builds **server-side tool functions** for 8 agents (PA + 7 COO-routed operational agents) on the LangGraph server. No new frontend components, pages, or React code is created in this phase. The frontend generative UI that renders these tool outputs is Phase 17.

**What this UI-SPEC governs:**
- Copywriting contract for all tool output messages, error messages, and HITL approval descriptions produced by Phase 15 tools
- Data shape contracts that downstream generative UI (Phase 17) will consume
- Consistency with established tool output patterns from Phases 13-14

**What is NOT in scope:**
- New React components (Phase 17)
- Layout or spacing changes (no frontend code)
- Color or typography changes (no frontend code)

## Tool Output Data Shapes

These data shape contracts ensure Phase 17's generative UI can render tool outputs consistently.

### Standard Tool Return Shape

Every Phase 15 tool returns one of:

```typescript
// Success with data
{ data: T; message: string; count?: number }

// Success with no data
{ data: null; message: string }

// Error
{ error: string; message: string }

// HITL pending
{ pending: true; action: string; description: string; payload: Record<string, unknown> }

// HITL cancelled
{ cancelled: true; message: string }
```

### Chart-Ready Data (Data Analyst only)

```typescript
{
  chartType: "bar" | "line" | "pie" | "area";
  data: Array<Record<string, string | number>>;
  xKey: string;
  yKey: string;
  title: string;
}
```

This shape maps directly to Recharts props in Phase 17 (GUI-03).
