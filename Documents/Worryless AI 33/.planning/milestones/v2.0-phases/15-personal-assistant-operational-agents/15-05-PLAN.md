1. **`crossFunctionalQuery(userId: string, queryType: string): Promise<QueryResult>`**
   - Use the exact QUERY_TEMPLATES pattern from 15-RESEARCH.md "Data Analyst Cross-Functional Query" section
   - Pre-defined safe SQL templates ONLY (never LLM-generated SQL):
     - `revenue_by_month`: SELECT date_trunc('month', date) AS month, SUM(amount) AS total FROM public.transactions WHERE user_id = $1 AND type = 'income' GROUP BY 1 ORDER BY 1
     - `expenses_by_category`: SELECT category, SUM(amount) AS total FROM public.transactions WHERE user_id = $1 AND type = 'expense' GROUP BY 1 ORDER BY 2 DESC
     - `leads_by_status`: SELECT status, COUNT(*) AS count, COALESCE(SUM(deal_value), 0) AS total_value FROM public.leads WHERE user_id = $1 GROUP BY 1
     - `invoice_summary`: SELECT status, COUNT(*) AS count, SUM(amount) AS total FROM public.invoices WHERE user_id = $1 GROUP BY 1
     - `posts_by_platform`: SELECT platform, COUNT(*) AS count, AVG(engagement_likes) AS avg_likes FROM public.social_posts WHERE user_id = $1 GROUP BY 1
     - `support_ticket_summary`: SELECT status, priority, COUNT(*) AS count FROM public.support_tickets WHERE user_id = $1 GROUP BY 1, 2
     - `project_status`: SELECT status, COUNT(*) AS count FROM public.projects WHERE user_id = $1 GROUP BY 1

**Create `src/tools/operations/project-tools.ts`** with 5 functions:

1. **`createProject(userId: string, name: string, description?: string, startDate?: string, dueDate?: string): Promise<{ projectId: string; message: string }>`**
   - INSERT into public.projects RETURNING id
   - Message from UI-SPEC: `"Project created: \"{name}\""`

2. **`addMilestone(userId: string, projectId: string, title: string, dueDate?: string, owner?: string): Promise<{ milestoneId: string; message: string }>`**
   - Verify project belongs to user first: SELECT id FROM projects WHERE id=$1 AND user_id=$2
   - INSERT into public.project_milestones RETURNING id
   - Message from UI-SPEC: `"Milestone added to \"{projectName}\": \"{milestoneTitle}\""`

3. **`listProjects(userId: string, status?: string): Promise<{ projects: ProjectRow[]; summary: ProjectStatusSummary; message: string }>`**
   - SELECT * FROM public.projects WHERE user_id=$1 [AND status=$2] ORDER BY created_at DESC
   - Calculate ProjectStatusSummary from results
   - Empty message from UI-SPEC: `"No projects found."`
   - Summary message: `"{activeCount} active, {planningCount} planning, {onHoldCount} on hold, {completedCount} completed"`

4. **`trackMilestones(userId: string, projectId: string): Promise<{ milestones: MilestoneRow[]; message: string }>`**
   - SELECT pm.* FROM project_milestones pm JOIN projects p ON pm.project_id = p.id WHERE p.user_id=$1 AND pm.project_id=$2 ORDER BY pm.due_date
   - Allow updating status: if a milestoneId and newStatus are provided, UPDATE and return updated list

5. **`analyzeBottlenecks(userId: string): Promise<BottleneckAnalysis>`**
   - SELECT pm.title, pm.due_date, pm.owner, p.name as project_name FROM project_milestones pm JOIN projects p ON pm.project_id = p.id WHERE p.user_id=$1 AND pm.status='blocked'
   - Message from UI-SPEC: `"{blockedCount} blocked milestone(s) found across {projectCount} project(s)."` or `"No bottlenecks detected. All milestones are on track."`

**Create `src/tools/operations/process-tools.ts`** with 1 function:

   - System prompt: "Draft a Standard Operating Procedure. Be specific and actionable. Each step should be clear enough for someone unfamiliar with the process."
   - Message from UI-SPEC: `"SOP drafted: \"{title}\""`
